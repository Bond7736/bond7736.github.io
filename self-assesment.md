To gain a varied skillset, I devoted time to numerous projects during my degree. Each course contributed a different needed concept. CS 320 (Software Test, Automation QA) and CS 405 (Secure Coding) greatly contributed to my understanding of secure practices through the creation of JUnit tests, a full security policy, and analysis of vulnerable script files (numeric overflow, SQL injection, etc.). This culminated in a full-stack vacation booking application with database connection, and an advising assistance program using a vector to store and retrieve course prerequisites from csv files. To conclude this section, CS 410 (Software Reverse Engineering) yielded me a wider perspective on the importance of software security, alongside legacy system maintenance. These experiences represent the practical path of my learning experience. 

## My Choice of Artifact and Rationale

To demonstrate my growth as an emerging CS graduate, I’ve chosen to enhance the artifact “Advising_Assistance_Program.cpp” from the course CS 300. Originally it was a simple menu loop that allowed users to choose between loading a csv file, printing courses alphanumerically, and printing course prerequisites. To expand its functionality, I’ve made enhancements across three core categories while adhering to course outcomes, said categories being software engineering, algorithms/data structure and databases. The finalized artifact is an advising CRUD application. It features creation of student records, optimized searching of records via hash table, retrieval and deletion. Compared to the original program's algorithm (linear search via vector), the hash table improves efficiency by allowing for student record lookups and updates in O(1) constant time. Additionally the program connects to a local populated database with TDE enabled, enforced RBAC, and updates the hash table using a write-through methodology to simulate a cache. To align with course outcomes and improve as a developer, this work employs best practices and validation not previously seen in the original files. Choosing to enhance this artifact grants me the opportunity to see parallels between the SDLC/ADLC, demonstrate secure querying, and practice algorithmic problem solving, a skill that will serve me in my career and beyond.
